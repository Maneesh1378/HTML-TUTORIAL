inline vs block

In HTML, elements are categorized as inline, block, or inline-block based on how they behave in the document flow. Here’s a quick breakdown:

Block Elements:
Start on a new line and take up the full width available. Can contain other block elements and inline elements. Respect height, width, margin, and padding.
Examples: div,p,h1-h6,section,article,header,footer,nav,form,table,ul,ol.

Inline Elements
Do not start on a new line; they sit alongside other elements. Only take up as much width as necessary. Height and width settings don’t apply (margin/padding affects only horizontally).It Cannot contain block elements.
Examples: span,a,strong,em,b,i,u,img,br,small.

Inline-Block Elements
Behave like inline elements but allow height and width adjustments like block elements.
Examples: img,button,input,select.
